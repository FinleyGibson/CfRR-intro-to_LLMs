What is Machine Learning?

Traditional solution vs Machine Learning solution

Algorithm: Computational set of instructions to solve a problem or perform a task

Input -> Algorithm -> output

Traditional approach:
Write some rules using domain knowledge
 

 
 
 
 
Machine learning approach:
Apply no domain knowledge 
Supply data
Provide a framework for the rules
Let the computer adjust the rules until they fit the data

Neural Networks
Give us a generic way to define an algorithmic rule to solve a broad range of problems that:
Can take in any number of features
Can produce and number of outputs
Can be configured to any degree of flexibility by adding depth
Provides us a differentiable equation for our rule allowing us to set its parameters automatically, given sufficient data.

Downsides:
Lots of parameters means a lot of differentiating
Automatic setting of lots of parameters requires a lot of data, with known outputs
Inputs restricted to those of known and fixed size
Need numerical inputs and outputs

How is this any use when it comes to language? 
It is harder to comprehend to us humans but the computer doesn’t care.

Statelessnesss
 
 
```json
[
    { "role": "system", "content": "You are a helpful assistant." },
    { "role": "user", "content": "What is the capital of France?" },
    { "role": "assistant", "content": "The capital of France is Paris." },
    { "role": "user", "content": "What about Germany?" }
]
```
- The "memory" is just the growing list passed each time

Important concepts
Context Window
 
- The maximum number of tokens the model can see at once
- GPT-3.5: 4K tokens. GPT-4: 128K tokens. Gemini 1.5: 1M tokens
- Everything outside the window is **invisible** to the model
- Implication: long conversations eventually drop early context
- Implication: token cost = prompt tokens + response tokens
